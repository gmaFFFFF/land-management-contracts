# Договоры по землеустройству в AsciiDoctor
# Общие сведения
Необычный подход к организации хранилища проектов договоров -- создание и хранение их в формате AsciiDoctor.

Преимущества:
- Использование системы контроля версий (например, Mercurial)
- Переиспользование одного типового фрагмента договора (например, условие об ответственности сторон) в разных договорах
- Изменение типового фрагмента автоматически отображается во всех местах его применения
- Возможность переключаться между различными структурами договора (порядок и количество статей договора)
- Возможность добавлять комментарии и пояснения к отдельным условиям договора

Недостатки:
- Необходимо обучаться использовать данный подход
- Нет проверки орфографии
- Прежде чем увидишь, что получишь, нужно совершить много действий
- Пока неизвестен подход, позволяющий на выходе получать красиво оформленный Word файл


## Требования к установленному ПО
- [Java](https://www.java.com/ru/download/ie_manual.jsp?locale=ru)
- [Asciidoctorj](https://github.com/asciidoctor/asciidoctorj)
- [Pandoc](https://pandoc.org)

При наличии менеджера пакетов [chocolatey](https://chocolatey.org), установку можно запустить командой:

```choco install jre8 asciidoctorj pandoc --yes```

## Способ использования
- Исходные данные заносятся в файлы, расположенные в папке **.\data**
- Стиль выходного Word файла редактируется в файле **.\out\adoc2docx_style.docx**
- Запускается генератор **.\out\adoc2doxc.bat**
